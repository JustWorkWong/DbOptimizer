# Detailed Task Checklist

## Purpose

作为细分任务的标准入口文档，后续 AI 默认从这里读取任务。

## Standard Task Format

每个任务都必须包含：

1. `目标`
2. `前置依赖`
3. `新增类`
4. `新增方法`
5. `修改文件`
6. `前后端接口`
7. `数据库变更`
8. `验证方式`
9. `完成标准`

## Canonical Detailed Checklist

详细任务内容以：

- [IMPLEMENTATION_TASK_CHECKLIST.md](./IMPLEMENTATION_TASK_CHECKLIST.md)
- [TASK_CARDS_INDEX.md](./TASK_CARDS_INDEX.md)

为准。后续派发优先使用独立任务卡。

## Dispatch Rule

派任务时必须同时附带：

1. 当前任务条目
2. 对应 API 契约文档
3. 对应 workflow / frontend / data 设计文档
4. [AI_TASK_PACKET_TEMPLATE.md](./AI_TASK_PACKET_TEMPLATE.md)
